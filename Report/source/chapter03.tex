\chapter{Methods}%
\label{ch:methods}%

This chapter formalizes the dexterous grasping problem and presents the mathematical framework underlying the two-stage privileged teacher--student pipeline.
\Cref{sec:problem_formulation} states the problem as a partially observable decision process and gives the optimization objective.
\Cref{sec:privileged_teacher} presents the privileged teacher training, including the asymmetric actor--critic that exploits privileged state information.
\Cref{sec:distillation_framework} introduces the distillation framework: DAgger and its safety-aware variant SafeDAgger, and the visual encoder architecture used for RGB-based student policies.

\section{Problem Formulation}%
\label{sec:problem_formulation}%

Reinforcement learning problems are typically formalized as a \ac{MDP}, a mathematical model of sequential decision making under uncertainty~\cite{SuttonBarto2018}.
An \ac{MDP} is defined by the tuple $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$, where $\mathcal{S}$ is the set of possible \emph{states} of the world, $\mathcal{A}$ is the set of \emph{actions} the agent can take, $P : \mathcal{S} \times \mathcal{A} \to \Delta(\mathcal{S})$ is the \emph{transition function} that gives the probability distribution over the next state given the current state and action, $R : \mathcal{S} \times \mathcal{A} \to \mathbb{R}$ is the \emph{reward function} that scores how good a state--action pair is, and $\gamma \in [0,1)$ is a \emph{discount factor} that determines how much future rewards are valued relative to immediate ones.
The defining property of an \ac{MDP} is the \emph{Markov property}: the next state depends only on the current state and action, not on the full history of past states; in other words, the current state is a sufficient summary of everything the agent needs to know about the past in order to make an optimal decision.
At each time step $t$, the agent observes the current state $s_t$, selects an action $a_t$ according to its \emph{policy} $\pi(a_t \mid s_t)$, receives a reward $R(s_t, a_t)$, and transitions to a new state $s_{t+1} \sim P(\cdot \mid s_t, a_t)$.
The agent's task is to find a policy that maximizes the expected sum of discounted rewards collected over time.

The dexterous grasping task studied in this thesis does not fit the standard \ac{MDP} model, because at deployment the robot cannot directly measure the full state of the world.
It only sees noisy and incomplete sensory measurements: joint encoders and camera images, but not, for example, the exact pose of the object or the contact forces at each fingertip.
Such problems are described by a \ac{POMDP}, which extends the \ac{MDP} with an explicit observation channel that mediates between the underlying state and what the agent actually perceives.
A \ac{POMDP} is defined by the tuple
%
\begin{equation}\label{eq:pomdp}
    \mathcal{M} = (\mathcal{S}, \mathcal{A}, \mathcal{O}, P, O, R, \gamma),
\end{equation}
%
which adds, on top of the \ac{MDP} elements, an \emph{observation space} $\mathcal{O}$ and an \emph{observation function} $O : \mathcal{S} \to \Delta(\mathcal{O})$ that returns the distribution over possible observations given the true (hidden) state.
The full state $s_t \in \mathcal{S}$ contains the joint configuration and velocities of the arm--hand system, the object pose, velocity, and physical properties, and contact information.
At deployment, the agent does not have access to $s_t$ directly; instead, it receives an observation $o_t \sim O(\cdot \mid s_t)$ consisting of proprioceptive measurements and camera images, and must choose its action based on this partial view alone.

The objective is to find a policy $\pi: \mathcal{O} \to \Delta(\mathcal{A})$ that maximizes the expected discounted return
%
\begin{equation}\label{eq:objective}
    J(\pi) = \mathbb{E}_{\pi}\left[\sum_{t=0}^{T} \gamma^t R(s_t, a_t)\right].
\end{equation}
%
Solving \cref{eq:objective} directly under partial observability is difficult because the policy must simultaneously learn \emph{what} to do (the manipulation skill) and \emph{how} to perceive (extracting task-relevant information from raw sensor data).
The privileged teacher--student paradigm decouples these two challenges by training the teacher policy $\pi^*$ on an information-complete observation function $O^{\text{priv}}$ that is only available in simulation, and subsequently distilling $\pi^*$ into a student policy $\hat{\pi}$ that operates on the realistic observation function $O^{\text{partial}}$ available at deployment.
The remainder of this chapter formalizes both stages.

\section{Privileged Teacher Training}%
\label{sec:privileged_teacher}%

The teacher is trained via \ac{PPO} on an asymmetric actor--critic architecture that exploits privileged state information at training time while keeping the actor restricted to the observations that will be available at distillation time.
\Cref{fig:teacher_training} illustrates the overall teacher training pipeline.

\subsection{Asymmetric Actor--Critic}%
\label{sec:asymmetric_ac}%

In an asymmetric actor--critic~\cite{Pinto2018}, the actor (policy) and critic (value function) receive different observation vectors.
Let $o_t^{\text{actor}}$ denote the teacher's actor observation and $o_t^{\text{critic}}$ the critic observation, where $o_t^{\text{critic}}$ contains additional privileged information beyond $o_t^{\text{actor}}$, such as object velocities and contact forces.
The actor parameterizes a Gaussian policy
%
\begin{equation}\label{eq:actor}
    \pi_\theta(a_t \mid o_t^{\text{actor}}) = \mathcal{N}\!\left(\mu_\theta(o_t^{\text{actor}}),\; \sigma_\theta^2\right),
\end{equation}
%
while the critic estimates the state--value function
%
\begin{equation}\label{eq:critic}
    V_\phi(o_t^{\text{critic}}) \approx \mathbb{E}_\pi\!\left[\sum_{k=0}^{T-t} \gamma^k R(s_{t+k}, a_{t+k}) \;\middle|\; s_t\right].
\end{equation}
%
This asymmetry allows the critic to use richer information for more accurate value estimates during training, while the actor learns a policy that depends only on the subset of state that will subsequently be available to the student through distillation.
In effect, the privileged information is consumed by the critic to shape gradients but never directly enters the deployed policy.

In the dexterous grasping scenario studied in this thesis, the asymmetry is particularly pronounced.
\Cref{tab:ac_observations} summarizes the observation components for actor and critic.
The critic observes the full simulation state, including per-body positions and velocities for the palm and all five fingertips (54D), object linear and angular velocity (6D), aggregate contact forces (3D), and measured joint torques for all 28 DOFs including coupled joints (23 actuated + 5 mimic), none of which are available on the real robot.
The actor, by contrast, receives only the palm center position and velocity (6D), proprioceptive joint measurements, and a scalar object identifier, which are the same inputs that the student will later receive alongside its RGB images.
Because the critic is discarded after training, the privileged information it consumes does not affect the deployed policy~\cite{Pinto2018}.
The teacher's actor therefore already operates on the same proprioceptive interface that will be available on the real robot, and distillation only needs to add visual perception on top.

Actions are scaled from the policy's normalized output $a_t \in [-1, 1]^{23}$ to absolute joint position targets using per-joint scaling factors.
The policy directly commands joint positions without geometric fabrics or motion primitives, which distinguishes this work from the FGP-based approach in DextrAH-RGB~\cite{Singh2025}.

\subsection{Reward Structure}%
\label{sec:reward_structure}%

The reward function decomposes as a signed, weighted sum of $K$ terms,
%
\begin{equation}\label{eq:reward}
    R(s_t, a_t) = \sum_{i=1}^{K} \operatorname{sign}_i \cdot w_i \cdot r_i(s_t, a_t), \quad \operatorname{sign}_i \in \{+1, -1\},\; w_i > 0,
\end{equation}
%
where each $r_i \geq 0$ is a non-negative per-term reward and $\operatorname{sign}_i$ determines whether the term acts as an incentive ($+$) or a penalty ($-$).
Positive terms encourage desired behavior, while negative terms penalize undesirable behavior.
\Cref{tab:reward_terms} summarizes the reward and penalty terms used in this thesis, including their functional forms and weights.
The lift and goal terms are gated by a contact mask $m_c$, ensuring the policy only receives reward for lifting when actually grasping the object rather than accidentally launching it.
Hand joints are penalized $1.5\times$ more than arm joints for action rate and $3\times$ more for joint velocity, encouraging smooth finger motions.

\subsection{Automatic Domain Randomization}%
\label{sec:adr_formulation}%

To produce a teacher that transfers across varying physical conditions, the training environment applies \ac{ADR}~\cite{OpenAI2020}.
Let $\boldsymbol{\xi} \in \Xi$ denote the vector of randomizable simulation parameters such as friction, mass, and actuator gains.
\ac{ADR} maintains an interval $[\xi_i^{\text{lo}}, \xi_i^{\text{hi}}]$ for each parameter and samples $\xi_i \sim \mathcal{U}(\xi_i^{\text{lo}}, \xi_i^{\text{hi}})$ at every episode reset.
Whenever the policy's rolling success rate exceeds a threshold $\tau_{\text{ADR}}$, a shared increment counter $n$ is advanced and each interval is interpolated from its initial range toward its maximum range,
%
\begin{equation}\label{eq:adr}
    \xi_i^{\text{lo/hi}}(n) = \xi_i^{\text{start, lo/hi}} + \frac{n}{N_{\text{max}}}\!\left(\xi_i^{\text{max, lo/hi}} - \xi_i^{\text{start, lo/hi}}\right),
\end{equation}
%
yielding an emergent curriculum from near-nominal physics to progressively more challenging conditions.
The curriculum uses $N_{\text{max}} = 50$ increments and a success threshold $\tau_{\text{ADR}} = 0.4$.
\Cref{tab:adr_params_methods} lists the key randomized parameters and their start/max ranges.
The maximum ranges for finger joint stiffness and damping correspond to values measured on the real Agile Hand hardware, and the thumb rotation velocity limit is ramped toward the hardware limit of approximately 12\,deg/s ($\approx 0.21$\,rad/s).

In addition to physics-parameter randomization, \ac{ADR} controls an external wrench perturbation applied to the grasped object.
Whenever the hand is within a proximity threshold of the object, random forces and torques are applied at each perturbation interval, with magnitudes scaled by the object mass.
The maximum linear acceleration is ramped from $0$ to $10\,\mathrm{m/s^2}$ over the \ac{ADR} schedule, forcing the teacher to develop robust force-closure grasps that resist progressively stronger perturbations.

\section{Policy Distillation}%
\label{sec:distillation_framework}%

Given a trained teacher $\pi^*$ operating on privileged observations, the goal of distillation is to train a student policy $\hat{\pi}_\theta : \mathcal{O}^{\text{partial}} \to \mathcal{A}$ that imitates $\pi^*$ using only the partial observations available at deployment.
This section presents the distillation algorithms (DAgger and its safety-aware variant SafeDAgger) and the visual encoder architecture used when the student operates on RGB images.
\Cref{fig:student_training} illustrates the overall student distillation pipeline.

The frozen teacher policy provides teacher actions $\boldsymbol{a}_T$ from noisy state observations, while the student policy produces actions $\boldsymbol{a}_S$ from stereo RGB images and proprioception. The safety policy evaluates the inverse-variance-weighted L2 action disagreement between teacher and student to determine whether the student may act autonomously or the teacher must intervene, controlled by the safety threshold~$\delta$.}

\subsection{DAgger and SafeDAgger Based Distillation}%
\label{sec:dagger_method}%

\ac{BC} on a fixed dataset of teacher trajectories suffers from compounding errors: small deviations from the teacher's state distribution lead the student into unvisited states, causing the policy to drift and fail~\cite{Ross2011}.
DAgger~\cite{Ross2011} addresses this by iteratively collecting data under the student's own policy while labelling it with the teacher's actions.
However, DAgger does not prevent the student from visiting dangerous states during data collection, which is unsatisfactory for dexterous hands where unsafe actions can damage hardware.

This thesis therefore adopts SafeDAgger~\cite{Zhang2016}, which augments DAgger with a safety gate.
Since the teacher is queried at every state to label the dataset, the learned safety classifier of Zhang and Cho~\cite{Zhang2016} is redundant; the action disagreement can be computed directly.
Let
%
\begin{equation}\label{eq:safety_label}
    e_t \;=\; \sum_{i=1}^{\dim \mathcal{A}} \frac{\bigl(\mu_{\theta,i}(o_t^{\text{student}}) - \mu^*_i(o_t^{\text{teacher}})\bigr)^2}{\sigma^{*\,2}_i(o_t^{\text{teacher}})}
\end{equation}
%
denote the per-step inverse-variance-weighted L2 action disagreement, where $\mu_\theta$ and $\mu^*$ are the mean actions of the student and teacher and $\sigma^{*\,2}_i$ is the teacher's predicted variance for action dimension $i$.
This is the same per-step quantity used as the action imitation loss in \cref{eq:wl2_loss}, so SafeDAgger's safety gate and the supervision signal share a single scale.
The executed action is selected by thresholding $e_t$ against a safety threshold $\delta > 0$:
%
\begin{equation}\label{eq:safedagger_intervention}
    a_t = \begin{cases}
        \hat{\pi}_\theta(o_t^{\text{student}}) & \text{if } e_t \leq \delta \quad (\text{student acts}), \\
        \pi^*(o_t^{\text{teacher}}) & \text{otherwise} \quad (\text{teacher intervenes}).
    \end{cases}
\end{equation}
%
Setting $\delta \to \infty$ recovers vanilla DAgger; $\delta \to 0$ recovers pure \ac{BC}.
The threshold $\delta$ is calibrated empirically so that roughly $20$--$30$\,\% of environments are flagged unsafe at the start of distillation; the intervention rate then decays naturally as the student improves, providing an implicit curriculum from teacher-guided to student-autonomous execution.
The calibration procedure is detailed in \Cref{sec:ablation_threshold}.
The complete procedure is given in \Cref{alg:safedagger}; \hlsafe{highlighted lines} mark the safety-gating additions relative to vanilla DAgger.

\begin{algorithm}[htb]
\caption{Online SafeDAgger with direct action-disagreement gating. \hlsafe{Blue lines} indicate the safety-gating modifications that distinguish SafeDAgger from vanilla DAgger.}
\label{alg:safedagger}
\begin{algorithmic}[1]
\State Initialize student $\hat{\pi}_{\theta}$, \hlsafe{safety threshold $\delta > 0$}
\For{each time step $t$ in rollout}
    \State Query teacher: $a_t^* = \pi^*(o_t^{\text{teacher}})$
    \State \hlsafe{Compute action disagreement $e_t$ as in \cref{eq:safety_label}}
    \If{\hlsafe{$e_t \leq \delta$}}
        \State \hlsafe{Execute $a_t = \hat{\pi}_{\theta}(o_t^{\text{student}})$} \Comment{\hlsafe{Student acts}}
    \Else
        \State \hlsafe{Execute $a_t = a_t^*$} \Comment{\hlsafe{Teacher intervenes}}
    \EndIf
    \State Update $\hat{\pi}_{\theta}$ on $(o_t^{\text{student}}, a_t^*)$ using \cref{eq:total_loss}
\EndFor
\State \Return $\hat{\pi}_{\theta}$
\end{algorithmic}
\end{algorithm}

Unlike classical DAgger, which accumulates an aggregated dataset across iterations and retrains the student on the full history, the implementation used here updates the student \emph{online} using only the current-step teacher label, without maintaining a replay buffer.

The student is optimized under a combined loss
%
\begin{equation}\label{eq:total_loss}
    \mathcal{L}(\theta) = \mathcal{L}_{\text{action}}(\theta) + \lambda_{\text{aux}}\,\mathcal{L}_{\text{aux}}(\theta),
\end{equation}
%
consisting of an action imitation term and an auxiliary regularization term.
The action loss is the \emph{inverse-variance-weighted L2 loss} from DextrAH-RGB~\cite{Singh2025}, in which each action dimension $i$ is rescaled by the teacher's per-dimension variance $\sigma^{*\,2}_i$:
%
\begin{equation}\label{eq:wl2_loss}
    \mathcal{L}_{\text{action}}(\theta) = \sum_{i=1}^{\dim \mathcal{A}} \frac{\bigl(\mu_{\theta,i}(o_t^{\text{student}}) - \mu^*_i(o_t^{\text{teacher}})\bigr)^2}{\sigma^{*\,2}_i(o_t^{\text{teacher}})},
\end{equation}
%
where $\mu_\theta$ and $\mu^*$ are the mean actions of the student and teacher.
Dimensions in which the teacher is confident (small $\sigma^*$) contribute more strongly, while uncertain dimensions contribute less, yielding an automatic weighting that does not require hand-tuning arm-versus-hand action scales.
The auxiliary loss $\mathcal{L}_{\text{aux}} = \|\hat{\mathbf{x}}_{\text{obj}} - \mathbf{x}_{\text{obj}}\|_2^2$ regresses the ground-truth object position from the visual encoder's representation, providing an additional learning signal that regularizes the encoder toward spatially meaningful features.

\subsection{RGB-Based Perception}%
\label{sec:rgb_perception}%

The student replaces the teacher's privileged object state with stereo RGB images.
This thesis reuses the visual encoder architecture from DextrAH-RGB~\cite{Singh2025}.

A stereo pair is used rather than a single camera because the student must infer the 3D position of the object without access to the privileged pose that the teacher saw; a stereo pair encodes parallax, which resolves the metric depth ambiguity of monocular images.
The left and right RGB frames are passed through a shared (Siamese) ResNet-18~\cite{He2016} backbone, producing $128$ tokens of dimension $128$ per view.
These tokens, together with a learnable \verb|[embed]| token, are fed into a two-layer transformer with a cross-attention mask that permits attention only \emph{between} views, forcing the network to learn implicit stereo correspondences without explicit depth supervision (\Cref{fig:stereo_full}).
The output of the \verb|[embed]| token is projected by an \ac{MLP} into a $64$-dimensional stereo embedding, which is concatenated with proprioception and fed into an \ac{LSTM} ($512$ hidden units) that provides temporal context across time steps.
The entire stack is optimized jointly under the inverse-variance-weighted L2 loss (\cref{eq:wl2_loss}).

In addition to the dynamics gap addressed by \ac{ADR}, the sim-to-real gap acquires a \emph{visual} component when the student operates on camera images.
Following DextrAH-RGB~\cite{Singh2025}, visual domain randomization is applied during distillation to mitigate this: at each episode reset, dome-light HDRI backgrounds, material properties (albedo, roughness, metallic constant) of the robot, table, and object, as well as object and table textures are randomized.
Additional data augmentations include color jitter and motion blur.
%
