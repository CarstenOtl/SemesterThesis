\chapter{Methods}%
\label{ch:methods}%

This chapter presents the methodology used in this work.
It describes the hardware platform, the simulation environment built in Isaac Lab, the privileged teacher policy trained via reinforcement learning, and the two distillation approaches---DAgger and SafeDAgger---used to transfer the teacher's behavior to a vision-based student policy.

%% =========================================================================
\section{System Overview}%
\label{sec:system_overview}%

The system follows a two-stage privileged teacher-student paradigm, illustrated in~\cref{fig:system_overview}.
In the first stage, a \emph{privileged teacher policy} is trained in simulation via Proximal Policy Optimization (PPO).
The teacher receives full state information---ground-truth object poses, joint positions, velocities, and contact forces---that would be unavailable on real hardware.
In the second stage, the teacher's behavior is \emph{distilled} into a vision-based student policy that operates from RGB images.
This distillation is performed using either DAgger or SafeDAgger, and the resulting student policies are compared in terms of task performance and training safety.

\section{Simulation Environment}%

All training and distillation is performed in NVIDIA Isaac Lab, a GPU-accelerated simulation framework built on Isaac Sim.
This section describes the environment design, asset integration, and key configuration choices.

\subsection{Isaac Lab Manager-Based Architecture}%
\label{sec:manager_based}%

Isaac Lab provides two paradigms for environment design: a \emph{direct} API and a \emph{manager-based} API~\cite{Mittal2023}.
This work uses the manager-based API, which structures the environment into modular components:

\begin{itemize}
    \item \textbf{Observation Manager:} Defines the observation spaces for both actor (policy) and critic (value function). The teacher receives privileged observations, while the student receives image-based observations.
    \item \textbf{Action Manager:} Maps policy outputs to joint position targets for the arm and hand.
    \item \textbf{Reward Manager:} Computes a weighted sum of reward terms at each time step.
    \item \textbf{Termination Manager:} Defines episode termination conditions (e.g., object dropped, time limit reached).
    \item \textbf{Curriculum Manager:} Optionally adjusts task difficulty during training.
\end{itemize}

\subsection{DAgger Distillation}%
\label{sec:dagger_distillation}%

The baseline distillation method follows the DAgger algorithm~\cite{Ross2011} as used in DextrAH-RGB~\cite{Singh2025}.
The procedure is as follows:

\begin{enumerate}
    \item Execute the current student policy $\hat{\pi}$ in the environment to collect trajectories.
    \item At each visited state $s_t$, query the privileged teacher $\pi^*$ for the expert action $a_t^* = \pi^*(s_t)$.
    \item Aggregate the new data $\{(o_t, a_t^*)\}$ with the existing dataset $\mathcal{D}$, where $o_t$ is the student's RGB observation.
    \item Retrain the student policy on the aggregated dataset $\mathcal{D}$ via supervised regression.
    \item Repeat for $N$ iterations.
\end{enumerate}

\subsection{SafeDAgger Distillation}%
\label{sec:safedagger_distillation}%

SafeDAgger~\cite{Zhang2016} extends DAgger by introducing a learned safety classifier that determines when the teacher should intervene during student rollouts.
This reduces unnecessary teacher queries and prevents the student from visiting dangerous states.

\subsubsection{Safety Classifier}%
\label{sec:safety_classifier}%

A binary classifier $D_\psi: \mathcal{S} \to \{0, 1\}$ is trained alongside the student policy.
The classifier predicts whether the student's action at a given state is ``safe'' (close to the teacher's action) or ``dangerous'' (far from it):
%
\begin{equation}\label{eq:safety_label}
    y_t = \begin{cases}
        1 & \text{if } \| \hat{\pi}(o_t) - \pi^*(s_t) \| \leq \delta \\
        0 & \text{otherwise}
    \end{cases}
\end{equation}
%
where $\delta$ is a threshold on the action disagreement.

\subsubsection{Teacher Intervention Mechanism}%
\label{sec:intervention}%

During data collection rollouts, the system uses the safety classifier to switch between student and teacher control:
%
\begin{equation}\label{eq:intervention}
    a_t = \begin{cases}
        \hat{\pi}(o_t) & \text{if } D_\psi(o_t) = 1 \text{ (safe)} \\
        \pi^*(s_t) & \text{if } D_\psi(o_t) = 0 \text{ (dangerous)}
    \end{cases}
\end{equation}
%
This mechanism creates an implicit curriculum: early in training, when the student is poor, the teacher intervenes frequently; as the student improves, it progressively takes over more of the rollout.

\section{Evaluation Protocol}%
\label{sec:eval_protocol}%

This section describes the metrics and procedures used to compare DAgger and SafeDAgger distillation.

\subsection{Performance Metrics}%
\label{sec:metrics}%

The following metrics are used to evaluate the distilled student policies:

\begin{description}
    \item[Grasp Success Rate (GSR):] Fraction of episodes where the student successfully grasps and lifts the object above a height threshold within the time limit.
    \item[Unsafe Episode Rate (UER):] Fraction of training episodes containing at least one unsafe event (e.g., object dropped from height, excessive joint torques, collision).
    \item[Teacher Query Rate (TQR):] Fraction of time steps where the teacher was queried during distillation (relevant for SafeDAgger).
    \item[Convergence Speed:] Number of distillation iterations required to reach a target success rate.
\end{description}
